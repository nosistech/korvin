# KORVIN Positioning

This page defines the public positioning for KORVIN.

Use this page as the source for README language, npm wording, GitHub About, release notes, and landing page copy.

## One-Line Positioning

KORVIN is an open-source, self-hosted AI agent framework for low-cost hardware.

Voice-first. Memory-persistent. Model-agnostic. No lock-in.

## Short Description

KORVIN helps users run their own AI agent foundation with local setup files, configurable model providers, persistent memory, and a path toward voice-first self-hosted operation.

The current public npm CLI focuses on safe local setup and checks. It does not deploy production services automatically.

## What KORVIN Is

KORVIN is open-source, self-hosted, model-agnostic, designed for low-cost hardware, built around persistent memory, designed for voice-first interaction, and intended to avoid provider lock-in.

## What KORVIN Is Not

KORVIN is not currently a turnkey VPS installer, Cloudflare setup wizard, Telegram setup wizard, LiteLLM runtime installer, dashboard service installer, systemd installer, provider-key wizard, public-port configuration tool, or managed SaaS product.

These may become future capabilities, but they should not be claimed until implemented, tested, and released.

## Who KORVIN Is For

KORVIN is for developers, homelab users, privacy-focused users, AI builders, small teams, and technical users who want a practical self-hosted AI agent foundation they can modify.

## Core Value

KORVIN gives users a path to run an AI agent they control: their infrastructure, model choices, provider keys, memory strategy, data boundaries, and deployment path.

## What Works Today

Current public surfaces include the npm package, CLI version commands, korvin init, korvin init --voice, korvin init --yes, korvin doctor, local setup generation, local setup validation, command documentation, configuration documentation, quickstart documentation, and protected deployment guidance.

## Current CLI Boundaries

The npm CLI is local-only.

It does not configure VPS setup, Cloudflare setup, Telegram setup, LiteLLM runtime setup, dashboard service setup, systemd services, provider keys, public ports, production deployment, or voice runtime dependencies.

## Claim Audit Rule

Before publishing any public page, check that users can do the claimed behavior today, npm latest includes it, README agrees, npm agrees, the landing page agrees, unfinished features are marked planned or experimental, and secrets or private infrastructure details are absent.
